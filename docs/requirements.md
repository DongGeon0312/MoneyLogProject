# 머니로그(MoneyLog) 요구사항 정의서

## 0. 개요
개인 가계부 웹 서비스. 로그인한 사용자가 수입/지출을 기록하고 카테고리별·월별 통계를 확인한다.
핵심 원칙: **내 데이터는 나만 접근한다 (인가)**

- 기본 트랙: 전원 필수, "동작하는 서비스를 배포까지 완주"
- 도전 트랙: 기본 트랙을 모두 완료한 뒤에만 진행

## 1. 유저 스토리

| ID | 유저 스토리 | 연결 기능 |
|----|------------|----------|
| US-1 | 사용자로서 이메일/비밀번호로 회원가입·로그인하고 싶다, 내 가계부를 나만 보고 싶어서. | F-01 |
| US-2 | 사용자로서 수입/지출을 등록하고 싶다, 돈의 흐름을 기록해야 해서. | F-03 |
| US-3 | 사용자로서 등록한 거래를 수정·삭제하고 싶다, 잘못 입력할 수 있어서. | F-03 |
| US-4 | 사용자로서 거래를 카테고리로 분류하고 싶다, 어디에 돈을 많이 쓰는지 알고 싶어서. | F-02 |
| US-5 | 사용자로서 이번 달 거래를 카테고리·타입으로 걸러보고 싶다, 원하는 항목만 빠르게 보고 싶어서. | F-04 |
| US-6 | 사용자로서 이번 달 총수입·총지출·잔액과 카테고리별 지출을 보고 싶다, 예산 상태를 한눈에 파악하고 싶어서. | F-05 |
| US-7 | 사용자로서 내 데이터에만 접근하고 싶다, 남이 내 가계부를 보거나 고치면 안 되니까. | F-06 |
| US-9 (도전) | 사용자로서 거래를 키워드/기간/금액으로 검색하고 CSV로 내보내고 싶다, 특정 지출을 빠르게 찾고 백업하고 싶어서. | F-13 |
| US-10 (도전) | 사용자로서 통계를 차트로 보고 싶다, 숫자보다 그래프가 한눈에 들어와서. | F-12 |

## 2. 기능 명세 (MoSCoW)

| ID | 기능 | 설명 | 우선순위 |
|----|------|------|---------|
| F-01 | 회원가입/로그인 | 이메일·비밀번호로 가입, 로그인 시 JWT(accessToken) 발급, 비밀번호는 BCrypt 저장 | Must |
| F-02 | 카테고리 관리 | 가입 시 기본 카테고리 시드 + 추가·수정·삭제 | Must |
| F-03 | 거래내역 CRUD | 수입/지출 등록·조회·수정·삭제 | Must |
| F-04 | 목록 조회(필터·페이징) | yearMonth·type·categoryId 필터 + page/size 페이징 | Must |
| F-05 | 월별 통계 | 이번 달 총수입·총지출·잔액, 카테고리별 지출 합계 | Must |
| F-06 | 인가(본인 데이터) | 모든 조회·수정·삭제는 로그인 사용자(user_id) 기준으로만 허용 | Must |
| F-07 | 입력 검증·전역 예외 | Bean Validation + 일관된 에러 응답 | Must |
| F-08 | 프론트 화면(필수 2종) | 로그인, 목록+등록 | Must |
| F-09 | Swagger 문서화 | springdoc-openapi로 API 문서 제공 | Should |
| F-10 | 배포(Docker+CI/CD+EC2) | Docker + GitHub Actions + AWS EC2 실접속 URL | Must |
| F-12 | 통계 시각화 | 카테고리 파이차트, 월별 추이 라인차트 | Could (도전) |
| F-13 | 검색·CSV 내보내기 | 키워드/기간/금액범위 검색, CSV 내보내기 | Could (도전) |
| F-14 | Refresh Token | 토큰 재발급 흐름 | Could (도전) |
| F-15 | 테스트·커버리지 | 단위/통합 테스트 + 커버리지 측정 | Could (도전) |
| F-16 | Jenkins·K8s·무중단 배포 | Jenkins 파이프라인, K8s 배포, 롤링 업데이트 | Could (도전) |
| F-11 | 월 예산·초과 경고 | Budget 엔티티, 카테고리/전체 예산 초과 경고 | Won't (이번엔 제외) |

## 3. 비기능 요구사항

| 분류 | 요구사항 | 구체적 기준 |
|------|---------|------------|
| 보안 | 본인 데이터만 접근 | 모든 조회/수정/삭제 쿼리는 `user_id = 로그인 사용자` 조건 필수 |
| 보안 | 비밀번호 보호 | BCrypt 해시 저장, 평문 저장 금지 |
| 보안 | 인증 방식 | `Authorization: Bearer {accessToken}` JWT 검증 |
| 성능 | 목록 페이징 | 거래내역 목록은 page/size 페이징 (전체 로딩 금지) |
| 성능 | 통계 응답 | DB 집계 쿼리로 계산 |
| 검증 | 입력 유효성 | amount > 0, transactionDate 필수, type은 INCOME/EXPENSE만 허용 |
| 품질 | 일관된 에러 응답 | `{success:false, code, message, data:null}` 통일 |
| 배포 | 실제 배포·접속 | AWS EC2 실접속 URL |
| 문서 | API 문서화 | Swagger UI |

## 4. 범위(Scope)

| 구분 | 기본 과제 범위 (In Scope) | 도전 과제 범위 (Stretch, 기본 완료 후 진행) | 이번엔 안 함 (Out of Scope) |
|------|--------------------------|------------------------------------------|----------------------------|
| 인증 | 이메일 회원가입/로그인, JWT 발급 | Refresh Token 재발급 | 이메일 인증, 비밀번호 찾기, 소셜 로그인 |
| 거래 | 수입/지출 CRUD, 월·카테고리·타입 필터, 페이징 | 키워드/기간/금액 검색, CSV 내보내기 | 반복 거래, 첨부 영수증 이미지 |
| 통계 | 월별 총수입·지출·잔액, 카테고리별 지출 | 파이/라인 차트 시각화 | 연간 리포트, 카테고리 예측 |
| 예산 | (없음) | (없음 - 이번 범위 제외) | 예산 설정/초과 경고 |
| 프론트 | 화면 2종(로그인/목록+등록) | - | 반응형·모바일 앱, 다크모드 |
| 배포 | Docker + GitHub Actions + EC2 | Jenkins, K8s, 무중단 배포 | 오토스케일링, 멀티리전 |
| 품질 | 입력 검증, 전역 예외, Swagger | 단위/통합 테스트 + 커버리지 | 성능 부하 테스트 |

## 5. 진행 원칙
- 기본 과제(F-01~F-10)를 100% 완료하고 EC2 배포까지 마친 뒤에만 도전 과제 착수
- 도전 과제 우선순위: 검색/CSV → 통계 시각화 → Refresh Token → 테스트/커버리지 → Jenkins/K8s
